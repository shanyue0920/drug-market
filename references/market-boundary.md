## 3. 市场协同接口（只给清单）

### 3.1 准入（→ 准入 skill）
| 接口项 | 市场做 | 准入做 |
|---|---|---|
| 政策环境解读(A5) | 解读DRG/VBP/医保对市场影响 | 执行医保申报/集采投标/挂网 |
| 定价输入(H7) | 给价值定位→定价区间建议 | 做ICER/谈判底线/定价执行 |
| 价值证据包(B5) | 整合临床+经济+患者价值 | 药经数据生成/HEOR分析 |

### 3.2 销售（→ 销售 skill）
| 接口项 | 市场做 | 销售做 |
|---|---|---|
| 销售工具包(F1) | 话术/DA/竞品应对卡/异议库 | 工具的终端使用 |
| 销售培训(F2) | 产品/竞品/策略培训 | 代表日常管理/带教 |
| 目标客户策略(F3) | 客户分级/覆盖策略建议 | 客户覆盖执行 |
| POA输入(F4) | 策略框架/核心信息/推广节奏 | POA执行/纯销追踪/终端管理 |

### 3.3 医学（→ 医学 skill）
| 接口项 | 市场做 | 医学做 |
|---|---|---|
| 推广KOL vs 医学KOL(D6) | 推广KOL（讲者/观念领袖/规模传播） | 医学KOL（科学交流/IIS/MSL管理） |
| 证据转化(D2) | 证据→推广信息矩阵（从循证三件套提取核心信息） | 证据生成（RWE/IIS/IV期/PASS） |
| 医学核心信息包 | 市场做推广转化 | 医学给3-5条Key Medical Messages（科学主张） |
| 推广物料审校 | 市场提供内容输入 | 医学做科学准确性审校（MLR的M） |

### 3.4 注册（→ drug-reg skill）
| 接口项 | 市场做 | 注册做 |
|---|---|---|
| 注册里程碑解读(D3) | 解读批件/挂网/进院里程碑对推广节奏的影响 | 注册申报/审评跟踪/发补回复 |
| 新适应症拓展(B3) | 市场识别机会人群→提需求 | 注册做新适应症申报路径 |

### 3.5 合规（→ 合规/法务 skill）
| 接口项 | 市场做 | 合规做 |
|---|---|---|
| 推广物料MLR | 市场提供内容输入+标注"待MLR审核" | MLR审查（Medical+Legal+Regulatory） |
| on-label/off-label | 市场标注推广信息限于on-label | 合规定义红线/审查违规 |
| 数字化合规 | 市场按合规要求设计数字化推广 | 合规审查数字平台内容/数据隐私 |

## 4. 非市场边界（外置/指路）

| 项目 | 归属 | 说明 |
|---|---|---|
| 纯医学证据生成(RWE/IIS/IV期/PASS) | 医学 skill | 市场调用证据，不生成证据 |
| 纯准入谈判(医保/集采/进院/药经) | 准入 skill | 市场给价值定位输入，不做谈判 |
| 纯销售执行(终端/纯销/POA执行/代表管理) | 销售 skill | 市场给工具+策略，不执行 |
| 纯注册申报(CTD/审评/发补/变更) | drug-reg skill | 市场解读里程碑影响，不做申报 |
| 纯合规审查(MLR/红线/违规处理) | 合规 skill | 市场提供内容输入，不做审查 |
| 患者支持项目(PSP) | 销售/医学 | 市场可参与设计，执行归销售/医学 |
| 供应链/CMC | 注册/供应链 | 市场不涉及 |

## 5. 协同机制

### 5.1 跨部门协作节点
- **年度计划**：市场牵头，准入/销售/医学/注册输入 → 市场整合。
- **上市准备**：市场牵头launch readiness，注册给里程碑、准入给定价、医学给KOL/证据、销售给渠道。
- **推广物料**：市场起草 → 医学审科学性 → 合规审合规性（MLR）→ 市场定稿。
- **竞品应对**：市场牵头策略，医学给证据对比，销售给一线情报。

### 5.2 信息流转
- 医学 → 市场：医学核心信息包（3-5条Key Medical Messages）+ 证据更新。
- 市场 → 销售：销售工具包 + 推广策略 + POA输入。
- 市场 → 准入：价值证据包 + 定价价值定位。
- 准入 → 市场：医保/集采状态更新 → 市场调整推广策略。
- 注册 → 市场：注册里程碑更新 → 市场调整上市节奏。
